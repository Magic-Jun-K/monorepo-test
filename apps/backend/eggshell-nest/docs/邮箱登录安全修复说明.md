# 邮箱登录安全修复说明

## 问题背景

原项目中存在两个安全隐患：
1. 邮箱首次登录即注册时，系统会自动生成并存储随机密码
2. 邮箱注册的用户可以通过账号密码方式登录，存在安全风险

## 修复方案

### 1. 修改邮箱登录逻辑
- 移除了邮箱首次登录时自动生成密码的逻辑
- 邮箱注册的用户密码字段设置为null
- 邮箱注册的用户只能通过邮箱验证码登录

### 2. 增强账号密码登录验证
- 在`validateUser`方法中添加密码存在性检查
- 在`verifyPassword`方法中添加null密码检查
- 在`LocalStrategy`中添加异常处理，返回特定错误码

### 3. 新增密码设置功能
- 添加`setPasswordForEmailUser`方法，允许邮箱注册用户设置密码
- 添加`canUserSetPassword`方法，检查用户是否可以设置密码
- 新增`/auth/set-password`接口，供前端调用
- 新增`/auth/can-set-password`接口，检查用户状态

### 4. 数据库迁移
- 创建迁移脚本`1700000000000-ClearPasswordForEmailUsers.ts`
- 清除所有邮箱注册用户的密码
- 提供回滚功能

## 使用说明

### 运行迁移脚本

```bash
# 运行迁移（清除邮箱用户密码）
npm run typeorm migration:run

# 如果需要回滚
npm run typeorm migration:revert
```

### 新增API接口

#### 1. 设置密码
```
POST /auth/set-password
Authorization: Bearer <token>
Content-Type: application/json

{
  "password": "新密码（至少6位）"
}
```

#### 2. 检查是否可以设置密码
```
GET /auth/can-set-password
Authorization: Bearer <token>

响应：
{
  "canSetPassword": true
}
```

### 前端集成建议

1. **用户登录后检查密码状态**
   ```javascript
   // 检查用户是否可以设置密码
   const response = await fetch('/auth/can-set-password', {
     headers: {
       'Authorization': `Bearer ${token}`
     }
   });
   const { canSetPassword } = await response.json();
   
   if (canSetPassword) {
     // 显示设置密码的提示或弹窗
     showSetPasswordPrompt();
   }
   ```

2. **设置密码功能**
   ```javascript
   async function setPassword(password) {
     const response = await fetch('/auth/set-password', {
       method: 'POST',
       headers: {
         'Content-Type': 'application/json',
         'Authorization': `Bearer ${token}`
       },
       body: JSON.stringify({ password })
     });
     
     if (response.ok) {
       // 密码设置成功
       alert('密码设置成功，现在可以使用账号密码登录了');
     } else {
       // 处理错误
       const error = await response.json();
       alert(error.message || '密码设置失败');
     }
   }
   ```

### 错误处理

#### 账号密码登录错误码
- `LOGIN_METHOD_NOT_ALLOWED`: 该账户只能通过邮箱验证码登录

#### 设置密码错误信息
- `用户不存在`: 用户ID无效
- `该账户已有密码，无法重复设置`: 用户已经有密码
- `只有邮箱注册用户才能设置密码`: 用户不是通过邮箱注册的

## 安全建议

1. **强制密码设置**
   - 建议在用户首次通过邮箱登录后，引导用户设置密码
   - 可以设置密码为可选功能，但提供明显的设置入口

2. **密码强度要求**
   - 当前要求密码至少6位
   - 建议增加密码复杂度要求（大小写字母、数字、特殊字符）

3. **登录方式提示**
   - 当邮箱注册用户尝试账号密码登录时，显示明确的错误信息
   - 提供切换到邮箱登录的选项

## 测试建议

1. **测试邮箱注册用户**
   - 使用新邮箱注册，确认密码字段为null
   - 尝试使用账号密码登录，应被拒绝
   - 设置密码后，应能使用账号密码登录

2. **测试普通用户**
   - 确认普通用户的账号密码登录不受影响
   - 确认普通用户不能调用设置密码接口

3. **测试迁移脚本**
   - 在测试环境运行迁移脚本
   - 确认现有邮箱用户的密码被清除
   - 测试回滚功能

## 注意事项

1. 迁移脚本会清除所有邮箱注册用户的密码，请确保在合适的时机执行
2. 执行迁移前，建议备份数据库
3. 迁移执行后，现有的邮箱注册用户需要重新设置密码才能使用账号密码登录
4. 前端需要相应地更新，以支持新的密码设置功能
