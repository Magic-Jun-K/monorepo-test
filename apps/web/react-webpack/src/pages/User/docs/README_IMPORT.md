# 用户导入功能说明

## 功能概述

用户导入功能允许管理员通过Excel文件(.xlsx/.xls)批量导入用户信息。该功能集成在用户管理页面中，提供了便捷的用户批量创建方式。

## 导入格式要求

### 支持的文件格式

- `.xlsx` (Excel 2007及以上版本)
- `.xls` (Excel 2003版本)

### 必填字段

| 字段名   | 列名     | 说明               | 示例     |
| -------- | -------- | ------------------ | -------- |
| username | username | 用户名，系统内唯一 | zhangsan |

### 可选字段

| 字段名 | 列名  | 说明                     | 示例                 |
| ------ | ----- | ------------------------ | -------------------- |
| email  | email | 邮箱地址，必须是有效格式 | zhangsan@example.com |
| phone  | phone | 手机号，支持中国大陆格式 | 13812345678          |

### 系统自动处理字段

以下字段**不需要**在Excel文件中提供，系统会自动处理：

| 字段           | 自动处理方式                 |
| -------------- | ---------------------------- |
| ID             | 系统自动生成唯一标识         |
| 创建时间       | 设置为导入时间               |
| 更新时间       | 设置为导入时间               |
| 用户状态       | 自动设置为"ACTIVE"（活跃）   |
| 用户角色       | 自动分配"普通用户"角色       |
| 是否超级管理员 | 自动设置为false              |
| 密码           | 暂不设置，用户首次登录需设置 |

## Excel文件格式示例

### 完整示例模板

```
username    | email                 | phone
------------|----------------------|-------------
zhangsan    | zhangsan@example.com | 13812345678
lisi        | lisi@company.com     | 13987654321
wangwu      |                      | 15612345678
zhaoliu     | zhaoliu@test.com     |
```

### 最简示例（仅必填字段）

```
username
---------
zhangsan
lisi
wangwu
zhaoliu
```

## 数据验证规则

### 1. 用户名验证

- **必填**：不能为空
- **唯一性**：系统内不能重复
- **格式**：无特殊限制

### 2. 邮箱验证（可选）

- 如果提供，必须是有效的邮箱格式
- 系统内必须唯一
- 可以为空

### 3. 手机号验证（可选）

- 如果提供，必须符合中国大陆手机号格式（1开头的11位数字）
- 系统内必须唯一
- 可以为空

## 导入流程

1. **准备Excel文件**：按照上述格式要求准备用户数据
2. **下载模板**：可在导入弹框中下载CSV模板参考
3. **选择文件**：在用户管理页面点击"导入用户"
4. **上传文件**：拖拽或点击选择Excel文件
5. **执行导入**：点击确定开始导入
6. **查看结果**：系统显示导入成功数量和失败详情

## 错误处理

### 常见错误及解决方案

| 错误类型       | 错误提示                   | 解决方案             |
| -------------- | -------------------------- | -------------------- |
| 缺少必填字段   | "第X行：用户名为必填项"    | 确保username列有值   |
| 用户名重复     | "第X行：用户名 XXX 已存在" | 使用不同的用户名     |
| 邮箱格式错误   | 邮箱格式验证失败           | 检查邮箱格式是否正确 |
| 手机号格式错误 | 手机号格式验证失败         | 使用11位数字手机号   |

### 导入结果类型

1. **完全成功**：所有用户都成功导入
2. **部分成功**：部分用户成功，部分失败
3. **完全失败**：所有用户都导入失败

系统会详细显示每种情况的具体信息。

## 注意事项

### 文件要求

- 单次导入建议不超过1000个用户
- 确保Excel文件编码为UTF-8（避免中文乱码）
- 文件大小建议控制在10MB以内

### 权限要求

- 只有管理员和超级管理员可以使用导入功能
- 普通用户无法访问此功能

### 安全考虑

- 导入的用户初始没有密码
- 用户需要通过其他方式（如邮箱验证）设置初始密码
- 系统会记录导入操作的审计日志

### 最佳实践

- 导入前先备份现有用户数据
- 建议分批导入大量用户数据
- 导入后及时通知用户设置密码
- 定期检查导入用户的状态和权限

## 技术实现

### 后端实现

- 使用SheetJS/xlsx库解析Excel文件
- TypeORM进行数据库操作
- 自动事务处理确保数据一致性

### 前端实现

- React + TypeScript实现
- antd Upload组件处理文件上传
- 实时显示导入进度和结果

### 数据流程

1. 前端上传Excel文件
2. 后端解析文件内容
3. 验证数据格式和唯一性
4. 批量插入数据库
5. 分配默认角色
6. 返回导入结果
